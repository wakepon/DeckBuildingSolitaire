# プロジェクト概要
トランプの「スピード」をベースにしたデッキ構築バトルブラウザゲーム

# 技術スタック
- React + TypeScript
- Tailwind CSS
- Vite（ビルドツール）
- React Hooks（状態管理）

# 作業の進め方

## 必ず守ること
1. **まず計画を立てる**: コードを書く前に、以下を明確にすること
    - 何を作るか
    - どのファイルを作成/変更するか
    - 実装の手順

2. **計画をコメントで共有**: 実装を始める前に、計画をIssueにコメントとして投稿する

3. **計画に沿って実装**: 承認を待たずに計画通り実装を進めてOK

## 計画のフォーマット例
```
## 実装計画

### 概要
[何を実装するか]

### 作成/変更するファイル
- `src/components/xxx.tsx` - [役割]
- `src/types/xxx.ts` - [型定義]

### 実装手順
1. [ステップ1]
2. [ステップ2]
3. [ステップ3]
```

# コーディングルール

## TypeScript/React
- コンポーネントは関数コンポーネントで実装
- 型定義は`src/types/`に配置
- ユーティリティ関数は`src/utils/`に配置
- カスタムフックは`src/hooks/`に配置

## ファイル構成

```
├── index.html
├── src/
│   ├── components/    # Reactコンポーネント
│   ├── types/         # 型定義
│   ├── utils/         # ユーティリティ関数
│   ├── hooks/         # カスタムフック
│   ├── App.tsx        # メインアプリケーション
│   ├── main.tsx       # エントリーポイント
│   └── index.css      # グローバルスタイル
├── package.json
├── tsconfig.json
└── vite.config.ts
```

## その他
- スマホ対応（タッチ操作）を考慮
- GitHub Pagesで動作すること
- コメントは日本語でOK

## 注意事項
- 1ファイルが300行を超えたら分割を検討

## 開発コマンド
- `npm run dev` - 開発サーバー起動
- `npm run build` - プロダクションビルド
- `npm run preview` - ビルド結果のプレビュー
